## 阶段 A: 后端基础设施(任务 1-5)

### 任务 1: 后端项目骨架 + 虚拟线程配置
**目标**: Spring Boot 3.2 + JDK 21 工程可启动,虚拟线程开启

- 写 `backend/pom.xml`: spring-boot-starter-web 3.2.5, spring-boot-starter-data-jpa, mysql-connector-j, lombok, spring-boot-starter-test, jackson
- 写 `backend/src/main/resources/application.yml`: MySQL 连接 + API Key 占位 + `spring.threads.virtual.enabled: true`
- 写 `CityGlowApplication.java`
- **测试**: `CityGlowApplicationTests.contextLoads()` 启动成功(用 H2 test profile)
- **提交**: `chore(backend): scaffold spring boot 3.2 + jdk 21 + virtual threads`

### 任务 2: 数据库 schema.sql + JPA 实体
**目标**: 3 张表(users / observation_logs / astro_events)+ 实体类

- 写 `schema.sql`(文档 4.2 + 索引)
- 写 `entity/User.java`、`entity/ObservationLog.java`、`entity/AstroEvent.java` (@Entity)
- 写 `repository/UserRepository.java`、`ObservationLogRepository.java`、`AstroEventRepository.java`
- **测试**: 三个 Repository 的 `saveAndFind` 测试(H2)
- **提交**: `feat(backend): add jpa entities and repositories for 3 tables`

### 任务 3: StargazingIndex 算法(TDD)
**目标**: 观星指数计算工具类

- **先写测试** `StargazingIndexTest`:
  - 新月 + Bortle 1 + 云量 0 = 100
  - 满月 + Bortle 9 + 云量 100 = 0
  - 边界: 云量 20% 内不扣分
  - 消息映射: 85 → "今夜极佳!",60 → "适合观星"
- 跑测试看红
- 写 `util/StargazingIndex.java`
- 跑测试看绿
- **提交**: `feat(backend): implement stargazing index algorithm with tests`

### 任务 4: OpenWeatherClient + Record DTOs
**目标**: 封装 OpenWeatherMap 调用,Record DTO

- 写 Record DTOs: `OpenWeatherResponse`、`OpenWeatherAstronomy`、`AstroData`、`ForecastResult`
- 写 `OpenWeatherClient.java`(用 RestClient):
  - `getCurrentWeather(lat, lng)` → 云量
  - `getAstronomy(lat, lng, date)` → 日出日落月相
- **测试**: 用 MockRestServiceServer mock 第三方响应,断言解析正确
- **提交**: `feat(backend): add openweather client with record dtos`

### 任务 5: AstroForecastController + ForecastService(虚拟线程)
**目标**: `GET /api/v1/astro/forecast?lat=&lng=` 工作

- 写 `ForecastService.java`: 用 `Executors.newVirtualThreadPerTaskExecutor()` 并行调 OpenWeatherClient 两个端点,合并后调 StargazingIndex
- 写 `AstroForecastController.java`
- 统一响应 `ApiResponse<T>` 包装类
- **测试**: `AstroForecastControllerTest` MockMvc 断言 200 + 字段
- **提交**: `feat(backend): add astro forecast endpoint with virtual threads`

## 阶段 B: 后端事件 + 明信片(任务 6-8)

### 任务 6: AstroEvent 种子数据 + Controller
**目标**: `GET /api/v1/events` 返回时间轴事件

- 写 `data.sql` 预填 2026 主要天文事件(英仙座流星雨、月食、行星冲日等)
- 写 `AstroEventController.java`: `GET /api/v1/events`、`GET /api/v1/events/{id}`
- **测试**: controller 测试断言至少 5 条事件返回
- **提交**: `feat(backend): add astro events seed data and endpoints`

### 任务 7: NasaApodClient + Controller
**目标**: `GET /api/v1/apod` 返回每日天文一图

- 写 `NasaApodClient.java`(RestClient)
- 写 `ApodController.java`(带简单缓存: 24h 内复用)
- **测试**: Mock 第三方响应断言解析
- **提交**: `feat(backend): add nasa apod endpoint`

### 任务 8: PostcardService + ObservationLogController(结构化并发)
**目标**: `POST /api/v1/logs/upload` 并行处理图片

- 写 `PostcardService.java`:
  - 用 `StructuredTaskScope.ShutdownOnFailure()` 并行三子任务:
    1. 图片压缩(ImageIO,长边 1920px)
    2. 水印绘制(Graphics2D)
    3. EXIF 元数据
  - 子任务 join 后合并
- 写 `ObservationLogController.java`:
  - `POST /api/v1/logs/upload` (multipart)
  - `GET /api/v1/logs` 列表
  - `GET /api/v1/logs/{id}` 详情
  - `GET /uploads/{filename}.jpg` 静态资源
- MIME 校验: image/jpeg, image/png
- **测试**: 上传测试图断言 200 + logId + cardUrl; 错误 MIME 断言 400
- **提交**: `feat(backend): add postcard generation with structured concurrency`

### 任务 11: DarkSkyLeaflet 地图组件
**目标**: Leaflet 暗色地图 + 光污染覆盖层 + 点击事件
