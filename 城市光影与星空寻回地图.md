### 📂 项目文档概览

- **项目名称**：城市光影（CityGlow）—— 基于 JDK 21 的夜间光污染监测与星空寻回地图
- **文档版本**：V1.0
- **项目性质**：JavaWeb 课程设计 / 创新实验项目
- **核心立意**：关注光污染与环境暗夜保护，利用地理信息系统（GIS）与多源数据聚合技术，构建一个“寻找最后星空”的沉浸式 Web 应用。

---

### 1. 引言

#### 1.1 编写目的

本文档旨在明确“城市光影”系统的功能需求、非功能需求、数据架构及交互逻辑。重点阐述如何利用 JDK 21 的新特性（如虚拟线程、Record 类、结构化并发）来解决多源气象数据聚合中的高延迟问题，为后续的编码实现提供详实依据。

#### 1.2 项目背景

随着城市化进程加快，光污染日益严重，星空在城市中逐渐消失。本项目旨在建立一个可视化的光污染监测与观星辅助平台。用户可以通过系统查询所在地的暗夜等级，获取精准的天文气象预报，并记录自己的观星足迹，从而唤起公众对暗夜保护的关注。

#### 1.3 技术栈定义

- **开发语言**：Java 21 (LTS)
- **后端框架**：Spring Boot 3.2.x
- **前端技术**：HTML5 + Leaflet.js (地图) + Chart.js (图表) + TailwindCSS
- **数据库**：MySQL 8.0
- **外部接口**：OpenWeatherMap API (气象数据)、NASA APOD API (天文图片)
- **构建工具**：Maven 3.9+

### 2. 总体描述

#### 2.1 用户角色

系统主要包含两种角色：**寻星者** 与 **暗夜守护者**。

- **寻星者**：浏览地图、查询观星指数、查看天文事件。
- **暗夜守护者**：上传观星照片、标记新的观测点、分享观测日志。

#### 2.2 运行环境

- **服务端**：JDK 21 运行时环境，支持 Docker 部署。
- **客户端**：现代浏览器（Chrome, Edge, Firefox），需支持 HTML5 Canvas 和 Geolocation API。

### 3. 功能需求详情

#### 3.1 模块一：暗夜地图

**功能描述**：系统的核心首页，以暗黑风格地图展示全球或城市的光污染分布。

- **详细需求**：
    - **光污染热力图**：基于 Bortle 暗空分类法（1-9级），在地图上通过不同颜色的覆盖层展示光污染程度（1级为极佳，9级为内城天空）。
    - **观星点标记**：地图上标记出“天文台”、“暗夜公园”、“最佳观星点”。
    - **交互逻辑**：点击地图任意位置，右侧滑出“环境详情面板”。
- **JDK 21 特性应用**：
    - 地图瓦片加载与数据聚合接口使用 **虚拟线程** 处理。当系统需要并发请求多个第三方气象 API（如云量、视宁度、湿度）来合成“观星指数”时，虚拟线程能有效避免平台线程的阻塞，显著提升高并发下的响应速度。

#### 3.2 模块二：观星条件预测面板

**功能描述**：为用户提供特定地点、特定时间的观星适宜度分析。

- **详细需求**：
    - **综合指数计算**：结合云量（<20%为佳）、月相（新月为佳）、光污染等级，计算出 0-100 的“今夜观星指数”。
    - **月相可视化**：使用 Canvas 动态绘制当前的月相（盈亏情况）。
    - **天文数据展示**：展示当天的日落时间、天文昏影终时间、日出时间。
- **JDK 21 特性应用**：
    - 使用 **Record 类** 封装气象与天文数据。例如 `public record AstroData(double cloudCover, int moonPhase, double bortleScale) {}`，利用 Record 的不可变性与简洁语法，确保数据传输对象的安全与高效。

#### 3.3 模块三：天文事件时间轴

**功能描述**：展示近期值得关注的天文现象。

- **详细需求**：
    - **事件列表**：以横向时间轴形式展示流星雨、月食、日食、行星冲日等事件。
    - **倒计时功能**：显示距离下一个重大天文事件的倒计时。
    - **详情弹窗**：点击事件，展示该天文现象的科普介绍及最佳观测方位。

#### 3.4 模块四：星空打卡与明信片

**功能描述**：用户的 UGC 内容模块，记录观星足迹。

- **详细需求**：
    - **照片上传**：用户上传拍摄的星空照片（JPG/PNG）。
    - **自动生成明信片**：系统自动在照片上合成水印，包含：拍摄地点（经纬度）、拍摄时间、当时的月相、Bortle 等级。
    - **分享功能**：生成带有唯一 URL 的星空明信片，可供他人浏览。
- **JDK 21 特性应用**：
    - 使用 **StructuredTaskScope**（结构化并发）处理明信片生成任务。并行执行“图片压缩”、“水印绘制”、“元数据写入”三个子任务，待所有任务完成后统一返回结果，优化处理耗时。

### 4. 数据需求

#### 4.1 核心实体关系

- **User** (1) ----< (N) **ObservationLog**
- **Location** (1) ----< (N) **ObservationLog**

#### 4.2 数据库表结构设计

**1. 用户表**

```sql
CREATE TABLE users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL,
    avatar_url VARCHAR(255),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**2. 观星日志表**

```sql
CREATE TABLE observation_logs (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT,
    location_name VARCHAR(100), -- 地点名称
    latitude DECIMAL(10, 7), -- 纬度
    longitude DECIMAL(10, 7), -- 经度
    image_url VARCHAR(255), -- 星空照片路径
    bortle_level INT, -- 当时观测到的暗夜等级
    description VARCHAR(500), -- 观测描述
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

**3. 天文事件表**

```sql
CREATE TABLE astro_events (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(100) NOT NULL, -- 事件标题（如：英仙座流星雨）
    event_time TIMESTAMP NOT NULL, -- 发生时间
    description TEXT, -- 事件描述
    event_type VARCHAR(20) -- 类型：METEOR, ECLIPSE, PLANET
);
```

### 5. 接口需求

#### 5.1 获取观星指数接口

- **URL**: `GET /api/v1/astro/forecast`
- **Request**: `lat` (纬度), `lng` (经度)
- **Response**: JSON

```json
{
  "code": 200,
  "data": {
    "score": 85,
    "cloudCover": 10,
    "moonPhase": "New Moon",
    "bortleLevel": 3,
    "message": "今夜适合观星！"
  }
}
```

- **逻辑**：接收坐标 -> 并行调用气象 API 与查询本地光污染数据 -> 计算综合得分 -> 返回。

#### 5.2 上传星空日志接口

- **URL**: `POST /api/v1/logs/upload`
- **Request**: `Multipart/form-data` (image, lat, lng, description)
- **Response**: JSON `{ "code": 200, "data": { "logId": 101, "cardUrl": "..." } }`

### 6. 非功能需求

#### 6.1 性能需求

- **并发能力**：利用 JDK 21 虚拟线程，系统需支持至少 500 个并发用户同时查询观星指数。
- **响应时间**：多源数据聚合接口响应时间 < 800ms；地图加载时间 < 1s。

#### 6.2 安全性

- **API 密钥保护**：第三方气象 API 的 Key 需存储在服务端环境变量中，严禁暴露在前端代码。
- **文件校验**：严格校验上传图片的 MIME Type，防止恶意文件上传。

#### 6.3 可用性

- **UI 风格**：全站采用深色模式（Dark Mode），模拟夜空视觉体验，减少屏幕光线对人眼暗适应的破坏。
